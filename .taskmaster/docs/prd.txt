## Problem Statement

실험실에서 파이펫(pipette) 조작은 반복적이고 정밀성이 요구되는 작업이다.
현재 이 작업을 자동화하는 Physical AI 시스템이 없으며, 로봇팔(Indy7)과 로봇손(Mark7)을
ROS2 환경에서 통합하고, 시연 데이터를 수집하여 모방학습 기반 자율 조작을 실현하는
end-to-end 파이프라인이 필요하다.

기존 문제:
- Mark7 로봇손에 대한 ROS2 공식 지원이 없음 (자체 개발 필요)
- Indy7 + Mark7 통합 제어 시스템 없음
- 파이펫 조작 데이터 수집 파이프라인 없음
- Physical AI 학습 및 배포 파이프라인 없음

## Target Users

**연구자/개발자 (1인 또는 소규모 팀)**
- 직접 교시(Indy7)와 키보드 조작(Mark7)으로 시연 데이터를 수집
- 수집된 데이터로 모방학습 모델을 훈련
- 훈련된 모델을 실제 로봇에 배포하여 자율 파이펫 조작 검증

## Success Metrics

- Mark7 드라이버: grasp/open/press 서비스 호출 시 500ms 이내 동작 완료
- 데이터 수집: 50 에피소드 이상 수집, 각 에피소드 15Hz 동기화 유지
- 모방학습: 최소 1개 프레임워크(LeRobot 우선)에서 파이펫 잡기 성공
- 자율 동작: 파이펫 거치대에서 파이펫 잡기 성공률 목표 (수치는 학습 후 결정)

</overview>

## Capability Tree

### Capability: Mark7 ROS2 Driver
Mark7 로봇손을 ROS2 Humble 환경에서 제어하기 위한 드라이버 및 관련 패키지 일체.
ros2_control 기반으로 하드웨어 추상화, 프리셋 서비스, 안전 모니터링을 제공한다.

#### Feature: Custom Message Types
- **Description**: GripperStatus, FingerState 커스텀 ROS2 메시지 타입 정의
- **Inputs**: 손가락 6개의 position/current/temperature 원시값
- **Outputs**: `pipet_hand_mark7_msgs/GripperStatus`, `pipet_hand_mark7_msgs/FingerState` 메시지 타입
- **Behavior**: 6개 손가락 인덱스 [Thumb Flex, Index, Middle, Ring, Little, Thumb Ab] 순서로 배열

#### Feature: Mark7 URDF / Description
- **Description**: Mark7 로봇손의 URDF 모델, 메시, RViz 설정 파일 작성
- **Inputs**: Mark7 물리적 사양 (DoF 구성, 링크/조인트 구조)
- **Outputs**: `pipet_hand_mark7_description` 패키지 (URDF, mesh, RViz config)
- **Behavior**: 손가락 6개 관절을 revolute joint로 모델링, robot_state_publisher와 연동

#### Feature: Serial Communication Layer
- **Description**: PC ↔ Mark7 동글 간 15바이트 Tx / 19바이트 Rx 직렬 통신 프로토콜 구현
- **Inputs**: 손가락 선택, 속도, 전류 한계, 목표 위치, 방향 파라미터
- **Outputs**: 15바이트 직렬 패킷 (Tx), 파싱된 손가락 상태 구조체 (Rx)
- **Behavior**: 헤더 0xAA 0x55 검증, Position=0은 "변경 없음" 처리, XOR 체크섬 검증, 중요 명령 2~3회 반복 전송

#### Feature: ros2_control Hardware Interface
- **Description**: ros2_control SystemInterface를 구현하여 Mark7 하드웨어를 ROS2 제어 프레임워크에 통합
- **Inputs**: URDF joint command (rad), `use_mock_hardware` 파라미터
- **Outputs**: `/joint_states` 토픽, `/gripper/status` 토픽 (20Hz)
- **Behavior**: `read()`에서 Rx 수신 → state 갱신, `write()`에서 command → Tx 패킷 전송; mock 모드에서는 command 값을 state로 복사

#### Feature: Grip Preset Service Node
- **Description**: grasp/open/press 프리셋을 std_srvs/Trigger 서비스로 제공
- **Inputs**: `/gripper/grasp`, `/gripper/open`, `/gripper/press` 서비스 요청
- **Outputs**: `/forward_position_controller/commands` 토픽 발행 (Float64MultiArray)
- **Behavior**: 각 프리셋에 대응하는 손가락 6개 목표 관절 각도(rad)를 ForwardCommandController에 발행; 캘리브레이션 완료 후 정확한 rad 값으로 업데이트

#### Feature: Safety Monitor Node
- **Description**: 손가락 온도/전류 이상을 감지하고 경고 및 긴급 정지
- **Inputs**: `/gripper/status` 토픽 (GripperStatus)
- **Outputs**: `/gripper/safety_warning` 토픽 (std_msgs/String), 임계값 초과 시 긴급 정지 명령
- **Behavior**: 온도 > 60°C 또는 전류 > 1100mA 시 경고 발행; 지속 시 ForwardCommandController에 현재 위치 유지 명령

#### Feature: Mark7 Keyboard Teleop
- **Description**: 터미널 키보드로 손가락 6개를 개별 제어하는 단독 테스트 노드
- **Inputs**: 키보드 입력
- **Outputs**: `/forward_position_controller/commands` 토픽 (Float64MultiArray)
- **Behavior**: 키 1회 입력 시 step 파라미터만큼 position count 변화; Thumb Flex 방향 이슈 해결 포함

#### Feature: Gazebo Simulation
- **Description**: Gazebo에서 Mark7를 가상 조작하고 실제 로봇에 미러링
- **Inputs**: Gazebo 마우스 조작, Mark7 URDF
- **Outputs**: Gazebo joint_states → `/forward_position_controller/commands`로 실제 로봇 미러링
- **Behavior**: gazebo_ros2_control 플러그인으로 가상 로봇 구동, Mirror Bridge 노드가 Gazebo joint_states를 실제 로봇 명령으로 변환

#### Feature: Joint Angle Calibration
- **Description**: URDF 관절 각도(rad)와 Mark7 position count 간 매핑 실측 및 기록
- **Inputs**: 실제 Mark7 하드웨어, 각 손가락별 물리적 가동 범위
- **Outputs**: 캘리브레이션 파라미터 파일 (YAML), GripPresetNode의 프리셋 값 갱신
- **Behavior**: 각 손가락을 단계적으로 이동시켜 rad ↔ position count 선형 매핑 측정

### Capability: System Integration Description
Indy7 + Mark7 + RealSense D435 전체 시스템의 통합 URDF 및 TF 트리 관리.

#### Feature: Integrated System URDF
- **Description**: Indy7 + Mark7 + RealSense D435를 하나의 URDF로 통합
- **Inputs**: indy7_ros2 URDF, pipet_hand_mark7_description URDF, RealSense D435 URDF
- **Outputs**: `pipet_system_description` 패키지, 통합 URDF, launch 파일
- **Behavior**: Mark7를 Indy7 엔드이펙터에 부착, RealSense를 Mark7와 함께 마운트; robot_state_publisher로 전체 TF 트리 관리

---

### Capability: Data Collection Pipeline
Indy7 직접 교시 + Mark7 키보드 조작 중 세 모듈의 데이터를 시간 동기화하여 NPZ 파일로 저장.

#### Feature: Multi-Modal Time Synchronization
- **Description**: /joint_states, RGB 이미지, Depth 이미지, /gripper/status를 ApproximateTimeSynchronizer로 ~15Hz 동기화
- **Inputs**: 4개 토픽 스트림 (20Hz, 30Hz, 30Hz, 20Hz)
- **Outputs**: 시간 정렬된 (joint, rgb, depth, gripper) 데이터 프레임
- **Behavior**: sync_slop=0.1초 허용 오차, 가장 느린 스트림(20Hz) 기준으로 동기화

#### Feature: Episode NPZ Recording
- **Description**: 녹화 시작~중지 구간의 동기화된 데이터를 에피소드 단위 NPZ 파일로 저장
- **Inputs**: 동기화된 데이터 프레임, /data_collector/start|stop 서비스 호출
- **Outputs**: `episodes/episode_<YYYYMMDD_HHMMSS>.npz` (timestamps, joint_positions, joint_velocities, joint_efforts, rgb_images, depth_images, gripper_actions)
- **Behavior**: RGB/Depth 이미지를 224x224로 리사이즈 후 저장; gripper_actions는 이산값 (0=유지/1=잡기/2=펴기/3=누르기)

#### Feature: Recording Control Service
- **Description**: 녹화 시작/중지를 서비스로 제어하고 현재 상태를 토픽으로 퍼블리시
- **Inputs**: `/data_collector/start`, `/data_collector/stop` 서비스 요청
- **Outputs**: `/data_collector/is_recording` 토픽 (std_msgs/Bool)
- **Behavior**: 이미 녹화 중에 start 요청 시 실패 반환, 녹화 중이 아닐 때 stop 요청 시 실패 반환

---

### Capability: Integrated Teleoperation
키보드 한 곳에서 Indy7 직접 교시 모드 제어, Mark7 그리퍼 서비스 호출, 데이터 수집 녹화 제어를 통합 수행.

#### Feature: Keyboard-to-Service Bridge
- **Description**: 키보드 입력을 모듈 서비스 호출로 변환하는 통합 텔레오프 노드
- **Inputs**: 터미널 키보드 입력
- **Outputs**: indy_srv(MOVE_HOME/DT_ON/DT_OFF), /gripper/grasp|open|press, /data_collector/start|stop 서비스 호출
- **Behavior**: SPACE=녹화 토글, H=홈, D=DT ON, d=DT OFF, G=잡기, O=펴기, P=누르기, Q=종료; 현재 녹화 상태를 터미널에 표시

---

### Capability: AI Training Pipeline
수집된 NPZ 에피소드 데이터를 각 학습 프레임워크 포맷으로 변환하고 모방학습 모델을 훈련.

#### Feature: NPZ to LeRobot Dataset Conversion
- **Description**: NPZ 에피소드 파일들을 LeRobot 데이터셋 포맷으로 일괄 변환
- **Inputs**: `episodes/*.npz` 파일 묶음
- **Outputs**: LeRobot 데이터셋 디렉터리 (HuggingFace datasets 포맷)
- **Behavior**: timestamps/joint_positions/rgb_images/gripper_actions를 LeRobot observation/action 스키마로 매핑

#### Feature: NPZ to HDF5 Conversion (Robomimic)
- **Description**: NPZ 에피소드를 Robomimic/Isaac Lab용 HDF5 포맷으로 변환
- **Inputs**: `episodes/*.npz` 파일 묶음
- **Outputs**: `robomimic_dataset.hdf5`
- **Behavior**: Robomimic의 데모 그룹 구조 (obs/actions/dones) 준수

#### Feature: NPZ to Unity Demo Conversion
- **Description**: NPZ 에피소드를 Unity ML-Agents GAIL 학습용 데모 포맷으로 변환
- **Inputs**: `episodes/*.npz` 파일 묶음
- **Outputs**: Unity demo 파일
- **Behavior**: Unity ML-Agents의 demonstration recorder 출력 포맷과 호환

#### Feature: LeRobot Model Training
- **Description**: LeRobot 프레임워크로 ACT 또는 Diffusion Policy 모방학습 모델 훈련
- **Inputs**: LeRobot 데이터셋, 하이퍼파라미터 설정
- **Outputs**: 훈련된 모델 체크포인트
- **Behavior**: LeRobot CLI 또는 Python API를 통한 훈련; 훈련 결과 로깅

#### Feature: Robomimic Model Training
- **Description**: Isaac Lab + Robomimic으로 BC(모방학습) → PPO(강화학습) 순서로 모델 훈련
- **Inputs**: HDF5 데이터셋, Isaac Lab 환경 설정
- **Outputs**: 훈련된 모델 체크포인트
- **Behavior**: BC로 초기 정책 학습 후 PPO로 강화; GPU 서버 필요

#### Feature: Unity ML-Agents Training
- **Description**: Unity ML-Agents의 GAIL + PPO로 시연 데이터 기반 모방학습
- **Inputs**: Unity demo 파일, ML-Agents 환경 설정
- **Outputs**: 훈련된 모델 (.onnx)
- **Behavior**: GAIL로 시연 모방 후 PPO로 정책 개선

### Capability: Autonomous Inference
학습된 모델을 실제 로봇에 배포하여 센서 입력으로 자율 파이펫 조작 수행.

#### Feature: Model Loading and Inference Node
- **Description**: 학습된 모델을 로드하고 센서 데이터를 입력받아 로봇 액션을 추론
- **Inputs**: RGB 이미지, Depth 이미지, Indy7 관절 상태, 모델 파일 경로, 모델 종류
- **Outputs**: Indy7 관절 궤적 명령 (FollowJointTrajectory 액션), Mark7 그리퍼 서비스 호출
- **Behavior**: ~15Hz 추론 루프; 연속 관절 제어는 토픽 방식, 그리퍼는 이산 서비스 호출

## Module Definitions

### Module: pipet_hand_mark7_msgs
- **Maps to capability**: Mark7 ROS2 Driver > Custom Message Types
- **Responsibility**: GripperStatus, FingerState 커스텀 메시지 타입 정의
- **File structure**:
  ```
  pipet_hand_mark7_msgs/
  ├── msg/
  │   ├── GripperStatus.msg
  │   └── FingerState.msg
  ├── CMakeLists.txt
  └── package.xml
  ```
- **Exports**:
  - `pipet_hand_mark7_msgs/GripperStatus` - 손가락 6개 상태 배열
  - `pipet_hand_mark7_msgs/FingerState` - 단일 손가락 상태 (position/current/temperature)

### Module: pipet_hand_mark7_description
- **Maps to capability**: Mark7 ROS2 Driver > Mark7 URDF / Description
- **Responsibility**: Mark7 URDF 모델 및 시각화 자원 제공
- **File structure**:
  ```
  pipet_hand_mark7_description/
  ├── urdf/
  │   └── mark7.urdf.xacro
  ├── meshes/
  ├── rviz/
  │   └── mark7.rviz
  ├── launch/
  │   └── display.launch.py
  ├── CMakeLists.txt
  └── package.xml
  ```

### Module: pipet_hand_mark7_driver
- **Maps to capability**: Mark7 ROS2 Driver > Serial Communication, Hardware Interface, Grip Preset, Safety Monitor
- **Responsibility**: Mark7 하드웨어 완전 제어 (통신, ros2_control, 프리셋 서비스, 안전 모니터링)
- **File structure**:
  ```
  pipet_hand_mark7_driver/
  ├── include/pipet_hand_mark7_driver/
  │   ├── mark7_hardware_interface.hpp
  │   ├── mark7_serial.hpp
  │   └── mark7_protocol.hpp
  ├── src/
  │   ├── mark7_hardware_interface.cpp   # ros2_control SystemInterface
  │   ├── mark7_serial.cpp               # Serial 통신 레이어
  │   ├── mark7_protocol.cpp             # 15/19바이트 패킷 인코딩/디코딩
  │   ├── grip_preset_node.cpp           # grasp/open/press 서비스 노드
  │   └── safety_monitor_node.cpp        # 온도/전류 감시 노드
  ├── config/
  │   ├── mark7_controllers.yaml         # ros2_control 컨트롤러 설정
  │   └── grip_presets.yaml              # grasp/open/press 관절 각도 값
  ├── launch/
  │   └── mark7.launch.py                # real/mock 모드 통합 런치
  ├── CMakeLists.txt
  └── package.xml
  ```
- **Exports** (ROS2 인터페이스):
  - Publishes: `/gripper/status` (GripperStatus, 20Hz), `/joint_states` (JointState, 20Hz)
  - Subscribes: `/forward_position_controller/commands` (Float64MultiArray)
  - Services: `/gripper/grasp`, `/gripper/open`, `/gripper/press` (std_srvs/Trigger)

### Module: pipet_hand_mark7_teleop
- **Maps to capability**: Mark7 ROS2 Driver > Mark7 Keyboard Teleop
- **Responsibility**: Mark7 단독 테스트용 키보드 제어 노드
- **File structure**:
  ```
  pipet_hand_mark7_teleop/

  │   └── keyboard_teleop_node.py
  ├── launch/
  │   └── teleop.launch.py
  ├── setup.py
  └── package.xml
  ```

## Dependency Chain

### Foundation Layer (Phase 0)
의존성 없음 — 가장 먼저 구축.

- **pipet_hand_mark7_msgs**: GripperStatus/FingerState 메시지 타입 제공. 다른 모든 Mark7 패키지가 의존.
- **pipet_hand_mark7_description**: Mark7 URDF 제공. driver, gazebo, system_description이 의존.
- **indy7_ros2**: 외부 서브모듈. 클론 및 빌드 확인만 필요. /joint_states, indy_srv 제공.
- **realsense2_camera**: 공식 패키지. apt 설치 및 동작 확인만 필요. /camera/.../image_raw 제공.

### Module Layer (Phase 1)
모듈 레이어 — 각 장치의 ROS2 인터페이스 구현.

- **pipet_hand_mark7_driver**: Depends on [pipet_hand_mark7_msgs, pipet_hand_mark7_description]
  - /gripper/status, /gripper/grasp|open|press, /joint_states 제공
- **pipet_hand_mark7_teleop**: Depends on [pipet_hand_mark7_driver]
  - Mark7 단독 테스트 가능
- **pipet_hand_mark7_gazebo**: Depends on [pipet_hand_mark7_driver, pipet_hand_mark7_description]
  - Gazebo 시뮬 + 미러링 제공
- **pipet_system_description**: Depends on [pipet_hand_mark7_description, indy7_ros2]
  - 통합 URDF/TF 제공

### Orchestrator Layer (Phase 2)
오케스트레이터 레이어 — 모듈 인터페이스 조합.

- **pipet_data_collector**: Depends on [indy7_ros2, realsense2_camera, pipet_hand_mark7_driver]
  - /data_collector/start|stop 서비스, NPZ 저장 제공
- **pipet_system_teleop**: Depends on [indy7_ros2, pipet_hand_mark7_driver, pipet_data_collector]
  - 통합 키보드 제어 제공

### Data Collection Operation (Phase 3)
실제 하드웨어 통합 및 에피소드 수집.

- **Joint Angle Calibration**: Depends on [pipet_hand_mark7_driver (real mode)]
- **System Integration Test**: Depends on [pipet_system_description, indy7_ros2, pipet_hand_mark7_driver, realsense2_camera]
- **Episode Collection (50+)**: Depends on [pipet_data_collector, pipet_system_teleop, Joint Angle Calibration, System Integration Test]

### AI Training Layer (Phase 4)
에피소드 데이터 변환 및 모델 학습.

- **ai/data_conversion**: Depends on [Episode Collection]
  - npz_to_lerobot, npz_to_hdf5, npz_to_unity 스크립트
- **ai/lerobot training**: Depends on [ai/data_conversion (npz_to_lerobot)]
- **ai/isaac_lab_robomimic training**: Depends on [ai/data_conversion (npz_to_hdf5)]
- **ai/unity_ml_agents training**: Depends on [ai/data_conversion (npz_to_unity)]

### Inference Layer (Phase 5)
학습된 모델 배포.

- **pipet_inference**: Depends on [indy7_ros2, realsense2_camera, pipet_hand_mark7_driver, ai/lerobot training OR ai/isaac_lab_robomimic training]

- [ ] LeRobot 백엔드 통합 (depends on: inference node, lerobot model)
  - Acceptance criteria: LeRobot 체크포인트 로드, 실시간 추론 15Hz 달성
- [ ] 실제 로봇 자율 동작 테스트 (depends on: inference node, real hardware)
  - Acceptance criteria: 파이펫 거치대에서 자율 파이펫 잡기 시도, 성공률 측정

**Exit Criteria**: 실제 로봇에서 자율 파이펫 잡기 동작 최소 1회 성공

**Delivers**: 전체 Physical AI 파이프라인 검증 완료

## Test Pyramid

```
        /\
       /E2E\       <- 10% (실제 하드웨어 + 전체 파이프라인 end-to-end)
      /------\
     /Integration\ <- 30% (ROS2 노드 간 토픽/서비스 연동)
    /------------\
   /  Unit Tests  \ <- 60% (패킷 인코딩/디코딩, 데이터 변환, 동기화 로직)
  /----------------\
```

## Coverage Requirements
- 핵심 모듈 (mark7_protocol, data_collector): 함수 커버리지 80% 이상
- ROS2 노드: integration test로 인터페이스 검증
- 하드웨어 의존 코드: mock 모드로 100% 커버

## Critical Test Scenarios

### Mark7 Serial Protocol (pipet_hand_mark7_driver)
**Happy path**:
- 15바이트 Tx 패킷 인코딩 → 기대값과 바이트 단위 일치
- 19바이트 Rx 패킷 디코딩 → 6개 손가락 position/current/temperature 정확 파싱

**Edge cases**:
- Position=0 입력 시 "변경 없음" 처리 (0으로 이동 명령 아님)
- 헤더 0xAA 0x55 불일치 패킷 수신 시 무시
- XOR 체크섬 불일치 시 패킷 드롭

**Error cases**:
- 시리얼 포트 없음 → 명확한 에러 메시지 + 노드 종료
- RF 신호 손실 (Rx 타임아웃) → 경고 로그, 마지막 상태 유지

**Integration points**:
- mock 모드: write() 후 read() 시 command 값이 state에 반영
- real 모드: grasp 서비스 호출 → /joint_states 변화

### Data Collector (pipet_data_collector)
**Happy path**:
- start → 10초 대기 → stop → NPZ 파일 생성, 모든 키 존재, shape 올바름
- rgb_images shape: (N, 224, 224, 3), dtype: uint8
- gripper_actions 범위: 0~3

**Edge cases**:
- 동기화 드롭 발생 시 프레임 스킵 (gap 없이 연속 저장)
- 빈 에피소드 (start 직후 stop) → 최소 1프레임 저장 또는 실패 반환

**Error cases**:
- stop 서비스 (녹화 중 아닐 때) → success=False 반환
- start 서비스 (이미 녹화 중) → success=False 반환
- 디스크 공간 부족 → 경고 로그

### GripPresetNode (pipet_hand_mark7_driver)
**Happy path**:
- /gripper/grasp 호출 → /forward_position_controller/commands에 grasp 관절값 발행
- /gripper/open 호출 → open 관절값 발행
- /gripper/press 호출 → press 관절값 발행

**Integration points**:
- ForwardCommandController가 명령 수신 후 /joint_states 업데이트

### Data Conversion Scripts (ai/data_conversion)
**Happy path**:
- npz_to_lerobot: 50개 NPZ → LeRobot 데이터셋, episode_index 연속, 타임스탬프 단조 증가

**Edge cases**:
- 손상된 NPZ 파일 → 건너뛰고 경고 로그
- 이미지 shape 불일치 → 리사이즈 후 저장

## Test Generation Guidelines
- ROS2 노드 테스트: `pytest` + `rclpy` + mock publisher/subscriber 패턴 사용
- 하드웨어 의존 테스트: `use_mock_hardware:=true` 파라미터로 분리
- 패킷 프로토콜 테스트: 독립 Python unittest (ROS2 불필요)
- AI 학습 테스트: 데이터 변환 스크립트는 소규모 (5 에피소드) 서브셋으로 빠른 검증

</test-strategy>

## Data Models

### Episode NPZ Schema
```
episode_<YYYYMMDD_HHMMSS>.npz
  timestamps:       (N,)         float64   # 녹화 시작 기준 상대 시간 (초)
  joint_positions:  (N, 6)       float32   # Indy7 관절 각도 (rad)
  joint_velocities: (N, 6)       float32   # Indy7 관절 속도 (rad/s)
  joint_efforts:    (N, 6)       float32   # Indy7 관절 토크 (N·m)
  rgb_images:       (N, 224, 224, 3) uint8 # RGB 이미지
  depth_images:     (N, 224, 224)   uint16 # Depth 이미지 (mm)
  gripper_actions:  (N,)         int8      # 0=유지/1=잡기/2=펴기/3=누르기
```

### GripperStatus Message
```
pipet_hand_mark7_msgs/GripperStatus
  std_msgs/Header header
  FingerState[6] fingers  # [Thumb Flex, Index, Middle, Ring, Little, Thumb Ab]

pipet_hand_mark7_msgs/FingerState
  float32 position      # position count (raw ÷ 4)
  float32 current       # 전류 (mA, raw ÷ 10)
  float32 temperature   # 온도 (°C)
```

## Technology Stack

| 항목 | 기술 |
|------|------|
| OS | Ubuntu 22.04 |
| ROS | ROS2 Humble |
| 로봇팔 드라이버 | neuromeka-robotics/indy-ros2 (서브모듈) |
| 로봇손 드라이버 | 자체 개발 (ros2_control 기반, C++) |
| 카메라 드라이버 | ros-humble-realsense2-camera (apt) |
| 오케스트레이터 | Python (rclpy) |
| AI 학습 #1 | Hugging Face LeRobot (ACT / Diffusion Policy) |
| AI 학습 #2 | NVIDIA Isaac Lab + Robomimic |
| AI 학습 #3 | Unity ML-Agents (GAIL + PPO) |
| 데이터 저장 | NumPy NPZ (수집), HDF5/LeRobot/Unity demo (학습) |

**Decision: ros2_control for Mark7 Driver**
- **Rationale**: 표준 ros2_control 인터페이스(ForwardCommandController 재사용)로 Gazebo 시뮬과 실제 하드웨어를 동일한 토픽/서비스로 제어 가능
- **Trade-offs**: 초기 설정 복잡도 증가 (URDF, controller yaml, plugin 등록)
- **Alternatives considered**: 직접 시리얼 통신 노드 (단순하나 ros2_control 생태계 활용 불가)

**Decision: 이산값 gripper_actions (0~3)**
- **Rationale**: 파이펫 조작은 잡기/펴기/누르기 세 동작으로 고정적 → 이산값으로 충분
- **Trade-offs**: 연속 손가락 제어 대비 표현력 제한 (추후 확장 고려)
- **Alternatives considered**: 손가락 6개 연속 관절 각도 (더 자연스럽지만 학습 복잡도 증가)

**Decision: ApproximateTimeSynchronizer (~15Hz)**
- **Rationale**: 서로 다른 주파수(20Hz, 30Hz)의 토픽을 단일 time-stamped 프레임으로 정렬
- **Trade-offs**: sync_slop=0.1초 허용 시 최대 100ms 시간 오차 발생 가능
- **Alternatives considered**: 최고 주파수(30Hz) 기준 보간 (복잡도 높음)

<risks>

## Technical Risks

**Risk**: Mark7 Rx 통신 미구현 (동글 Arduino 코드 미확보)
- **Impact**: High — /gripper/status 실시간 상태 피드백 불가
- **Likelihood**: Medium — 동글 코드 확보 시 해결 가능
- **Mitigation**: mock 모드에서 command → state 복사로 개발 진행; Rx 없이도 기본 제어 가능
- **Fallback**: Rx 없이 Tx 단방향 제어만으로 데이터 수집 진행; gripper_actions는 키 입력 이벤트로 대체 기록

**Risk**: Mark7 캘리브레이션 — URDF rad ↔ position count 매핑 불정확
- **Impact**: High — grasp/open/press 프리셋이 실제 파이펫 잡기에 부적합
- **Likelihood**: Medium — 실측 필요, 손가락별 비선형성 가능
- **Mitigation**: Phase 3에서 실측 후 grip_presets.yaml 업데이트; 반복 조정 허용
- **Fallback**: position count 직접 지정 방식(teleop)으로 먼저 최적값 탐색 후 rad 변환

**Risk**: RF 통신 불안정 (2.4GHz 신호 손실)
- **Impact**: Medium — 명령 누락으로 그리퍼 오동작
- **Likelihood**: Medium — 실내 환경에서 빈번히 발생 가능
- **Mitigation**: 중요 명령 2~3회 반복 전송, 타임아웃 감지 후 재전송 로직
- **Fallback**: 유선 방식 검토 (Mark7 제조사 문의)

**Risk**: AI 학습 데이터 부족 (50 에피소드 불충분)
- **Impact**: High — 모델이 수렴하지 않거나 일반화 실패
- **Likelihood**: Medium — 단순 작업이지만 Physical AI는 데이터 요구량 불명확
- **Mitigation**: 초기 50 에피소드로 학습 후 결과 보며 점진적 증가; 데이터 증강(이미지 augmentation) 적용
- **Fallback**: 데이터 수량보다 다양성 확보(거치대 위치/각도/조명 변화)에 집중

**Risk**: GPU 학습 서버 미확보
- **Impact**: High — Isaac Lab + Robomimic, Unity ML-Agents 학습 불가
- **Likelihood**: High — 현재 미확보 상태
- **Mitigation**: LeRobot을 우선 진행 (로컬 GPU 또는 Colab 활용 가능); GPU 서버 확보 후 병행
- **Fallback**: LeRobot 단일 프레임워크로 프로젝트 목표 달성

## Dependency Risks

**Risk**: indy7_ros2 서브모듈 API 변경 또는 빌드 실패
- **Impact**: Medium — Indy7 제어 전체 불가
- **Mitigation**: 특정 커밋 고정(git submodule pin), 별도 브랜치 유지
- **Fallback**: prev/indy7-direct-teaching 폴더의 검증된 버전 참조

**Risk**: realsense2_camera 패키지 Ubuntu 22.04 / ROS2 Humble 호환성 문제
- **Impact**: Medium — 카메라 데이터 수집 불가
- **Mitigation**: 공식 apt 패키지 (ros-humble-realsense2-camera) 사용으로 호환성 보장
- **Fallback**: librealsense SDK 직접 래핑 노드 작성

## Scope Risks

**Risk**: Gazebo 시뮬레이션 구현 우선순위 오버헤드
- **Impact**: Low — 핵심 경로(실제 하드웨어 데이터 수집)에 영향 없음
- **Mitigation**: Phase 1b로 분리, 핵심 Phase 완료 후 진행
- **Fallback**: Gazebo 미완성 시에도 전체 프로젝트 목표 달성 가능

**Risk**: 3가지 AI 학습 방법 동시 추진 분산
- **Impact**: Medium — 어느 방법도 충분히 검증 못 할 가능성
- **Mitigation**: LeRobot 우선 집중 (배포 친화적, GPU 요구 낮음), 나머지는 병행
- **Fallback**: LeRobot 단일 방법으로 프로젝트 완수

## Glossary
- **에피소드**: 홈 포지션 → 파이펫 잡기 1회 시연 단위
